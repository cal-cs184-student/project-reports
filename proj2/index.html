Overview

In this project we will dive into how Bezier Curve works and how to render and manipulate 3D meshes.

The project is divided into two sections. The first section focuses on Bezier curves and surfaces. Part one uses the de Casteljau algorithm to represent Bezier curves through 1D subdivision. The implementation of a simple Lerp function to iteratively evaluate the next step of the Bezier curve is discussed, and results are shown in the form of two animated gifs. Part two extends the de Casteljau algorithm to a 2D surface using separable 1D subdivision. The implementation is simplified by viewing the process as a two-step process, where 2D surfaces are sliced into multiple 1D curves and evaluated using the 1D de Casteljau algorithm. Results are shown in the form of two images.

The second section focuses on triangle meshes and half-edge data structures. Part three discusses area-weighted vertex normals and highlights their importance in computer graphics. The implementation of these normals is shown by iterating through the faces connected to the vertex and collecting all normals of the faces into a vector. The collected normals are then looped over and summed up weighted by their sizes to produce an average normal. Part four describes the implementation of edge flip, which involves re-linking all the assets associated with the flip. The debugging process is also discussed. In part five, edge split is implemented, and the debugging process is also discussed.

In this part, we will explore Bezier curves and their representation through de Casteljau's algorithm in 1D subdivision.

Implementation

Since this is only single dimensional, the algorithm is pretty straight-forward. We implemented a simple Lerp function (linear interpolation) to iteratively evaluate the next step of the Bezier curve. 

(1 - t) * points[i] + t * points[i + 1]

Part 2: Bezier Surfaces with Separable 1D de Casteljau

Implementation

Extending the de Casteljau Algorithm to a 2-D surface can be simplified greatly by viewing it as a two step process, where in the first step we slice the 2D surface in to multiple 1D curves to which we apply the 1D de Casteljau algorithm and get the Bezier coordinates. Then we can gather the array of point in the slices and evaluate a point across the slices using another 1D de Casteljau Algorithm, that point would be a point on the 3D Bezier Curve. This is a special case where all the points are quantized and on the exact 3D grid, making slicing really simple and workable.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

Area-Weighted Vertex Normals play a crucial role in computer graphics, particularly in the creation of realistic 3D models. These normals are a more accurate representation of the surface geometry of an object, since they take into account the surface area of each polygon in the mesh. This means that they can be used to create more precise lighting and shading effects, resulting in a more lifelike appearance. Additionally, area-weighted vertex normals can be used to smooth out the appearance of rough or jagged edges in a mesh, making it look more polished and professional.

Implementation

We simply iterate through the faces connected to the vertex using a Do-While loop and collect all normals of the faces into a vector.

std::vector<Vector3D> normals;
HalfedgeCIter currentEdge = halfedge();
FaceCIter currentFace = halfedge()->face();
do {
	normals.push_back(currentFace->normal());
	currentEdge = currentEdge->twin()->next();
	currentFace = currentEdge->face();
} while (currentFace != halfedge()->face());

We then loop over the collected normals and sum them up weighted by their sizes (which information is already contained in the normal vector’s length. 

for (int i = 0; i < normals.size(); i++) {
		avgNormal += normals[i];
}
return avgNormal.unit();

Implementation

  1. Compute the updated vertex positions associated with edges, and store it in Edge::newPosition.

    (1 - n * u) * original_position + u * original_neighbor_position_sum

  1. Flip any new edge that connects an old and new vertex.

Results and Discussion

Take some notes, as well as some screenshots, of your observations on how meshes behave after loop subdivision. What happens to sharp corners and edges? Can you reduce this effect by pre-splitting some edges?

As we can see, loop subdivision makes the surface much smoother. This is achieved by recursively dividing each face of the mesh into four smaller faces, and moving the vertices to a weighted average of their neighbors. This process is repeated a number of times, resulting in a much denser mesh with a smoother surface.

However, we can not keep any sharp corners and edges unless we define them as boundary edges. Pre-splitting the edges would not help either as seen below. This is because in absence of boundary, all edge will be averaged out thus no sharpness will remain. Furthermore pre-splitting would maybe yield an even smoother image because there are more triangles in the mesh to begin with.

As seen above, the edges are still very smooth. There does seem to have a better and more pronounced hexagonal hole in the middle, this may be due to the fact that the inner side is now split more symmetrically than before, leading to some good geometries.

  • Cube Without pre-processing

As we can see below the original cube with the diagonal mesh yields an asymmetric blob when conducting loop-subdivision. This is because in each subdivision, the geometry of the diagonals are averaged and rounded, thus if the original mesh is asymmetric the resulting one would be asymmetric as well.

  • Preprocessing the Cube (Making the mesh symmetric)

Pre-processing the cube by flipping and splitting edges can remove the asymmetry that occurs during loop subdivision. By making the mesh more symmetric through pre-processing, we achieve a more uniform subdivision. In order for the preprocess to work as expected, we need to split the diagonal edge on each face of the cube, thus creating four symmetric triangles on each side.
